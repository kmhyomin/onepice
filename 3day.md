>## 3강에서 배운것2
>1. useState를 쓰면서 값을 한개만 넣었을 때 => `useState('남자 코트 추천')` **첫번째는 실제자료**. 즉 '남자코트 추천' 이 들어있고, **뒤에는 함수가 남**는다. => ('남자 코트 추천',함수)
>
>2. 변수와 State의 차이점 <br>
>- **변수**는 내용물이 **갑자기 바꼈**을 때 (사용자의 수정, 혹은 내용물 추가 등), 변수는 자동으로 바뀌지 않음. => **html**에 **반영**되지 **않음**. (따로 변경 되었으니 수정하라는 코드를 짜야함.)<br>
>- **state**는 **그렇지 않음**. state를 쓰면 html이 **자동 재랜더링됨**. (반영)
>
> **빡대가리식 정리 : 자주 변경될 것 같은  html은 state 사용하기.**

## 4강에서 배운것
>- warning 메세지 지우는 법 -> `/* eslint-disable */` 
>- useState를 쓸 때 앞에 변수명을 작성하잖아. 그때 딱히 바뀔 예정이 없으면 두번째 함수는 안 써도 됨. (근데 일단 써두는 게 마음 편한 듯)
>- **`() => {}`** 이게 js function이랑 똑같은 기능을 함.
>- onClick 같은 이벤트 핸들러를 사용 할 땐 원하는 html 요소 옆에 붙여서 쓰면 된다. 
> <br> => `<span onclick={}> 아무말 <span>`
> 
>- ### State를 바꾸는 법. (*onclick 이벤트* 기반)
>1. 먼저 state를 2번째 변수까지 작성한다. <br>
> => `let [글자 , 바꾸기] =useState('안녕하세요.');`
>2. 바꾸고 싶은 곳에 함수를 쓴다. <br>
> => `<span onClick={() => {}}> {글자} <span>`
>3. 중괄호 안에 바꾸고 싶은 state의 2번째 변수명을 쓴 뒤 그 안에 변경할 대체값을 넣는다.
> => `<span onClick={() => {바꾸기("안녕")}}> {글자} </span>`
>4.  끝.
>- ### state에 숫자 더하는 법. (*onclick 이벤트* 기반)
>1. 기본값 설정 <br>
> => `let [숫자 , 더하기] =useState(0);`
>2. 바꾸고 싶은 곳에 함수를 쓴다. <br>
> => `<span onClick={() => {}}> {숫자} <span>`
>3. 중괄호 안에 두번째 작명을 쓴 뒤 그 안에 첫번째 작명 + 1을 쓴다. <br>
> => `<span onClick={() => {더하기(숫자 + 1)}}> {숫자} <span>`
## 5강에서 배운것
>- ### array에서 n번째 값만 바꾸는 법
>- array나 object를 쓸 땐 원본은 보존하는게 좋다. (영구적으로 바꿔버리면 나중에 원본 데이터가 필요할 때 못찾음.) => **카피 제작**하기
>1. 카피본 만들기 <br>
> => `let copy = 글제목;`
>2. 카피본을 수정하기 <br>
> => `copy[n] = '바뀐 글'`
>3. 두번째 작명(카피본 넣기)
> => `변경(copy);`
>4. 이렇게 하면 안됨. (ㅋㅋ)
>5. `let copy = 글제목;` 이곳에서 첫번째 작명에 [...첫번째 작명] 을 붙인다. <br>
> => `let copy = [...글제목];`
>> ### 동작 원리
>>- state 변경 함수는 변경 전 기존 state와 새 state를 비교해서 **둘이 같다면 변경하지 않는다**. (에너지 절약 차원)
>>- array, object의 동작방식이 특이 한것도 한 몫 하는데, <br>
>> 예를 들어 `let arr = [1,2,3]` 이란 array가 있다고 해보자. js에선 arr이란 변수 안에 123이 있는게 아니라 123을 미지의 공간(ram)에 저장시켜두고, arr이란 변수에는 찾아가는 이정표만 저장해둔다. <br> 그래서 **내용물이 바뀌는 말든 알빠 아님**. 내용물이 바뀐건 바꿔준다만 **이정표가 바뀌지 않는 이상  기존state랑 새state랑 바뀐게 없다고 판단 해 변경이 안됨.**
>>- `...`문법이 뭐냐면 array의 괄호를 벗겨주세요, 그 다음에 다씨 씌워주세요. 라는 거다.<br>
>>이러면 이정표도 달라져서 변경이 가능하다.
>>
>> **빡대가리식 정리 : state가 array 혹은 object라면 shallow copy(독립적 카피본)를 하나 만들어서 수정해야함. (정보 보존 + 이정표 바꾸기)**

>> => `let copyy = [...글제목];`

>>3. 변경시키기 <br>
